Object oriented programming

Module Overview

Object-Oriented Programming (OOP), a popular approach to software development that uses objects and classes to organize code. Students will learn how to create classes and objects in Python, and use them to manipulate data. The module covers advanced topics like inheritance and polymorphism, which are used to design complex programs. Upon completion, students will have the skills to design and implement their own OOP programs.

Learning outcomes

  • Appraise and critically evaluate object-oriented programming compared to other programming paradigms.
  • Design and implement programs that demonstrate appropriate use of object-oriented design principles.
  • Select and use appropriate data structures for a given problem.
  • Propose object-oriented solutions using an appropriate modelling language, such as UML.

    • Unit 1: An Introduction to Python Programming and the OO Programming Paradigm

    The first unit of the module introduces the object oriented programming paradigm, the syntax and the method used to form classes in python. Also introducing the 5 major features of object oriented programming:

  • Classes: OOP is based on the group of classes which are templates or blueprints that define the properties and behaviour of an object.
  • Encapsulation: is a way to hide the implementation details of an object from the outside. In OOP, data and methods are encapsulated within the class and are accessed by the methods.
  • Inheritance: allows for a new class to be based on an existing one. Inheriting the properties and methods. This allows for code reuse and helps to organize the code into a hierarchy.
  • Polymorphism: allows objects to change to take on different forms or types
  • Abstraction: is the process of removing unnecessary detail from an object, leaving only the essential characteristics.

    Artefacts

    UML models, Software Development Life Cycle

    Unified modelling language or UML is a modelling technique the illustrates a detailed development process of a system (Jacobson et al. 2004). In other words, UML is a general-purposed language that tends to support a step-by-step process to achieve a practical system. UML practices is considered a good development process and is crucial to the success of a software. Therefore, there are several types of UML models thar are used in different stages of the development process.

    1. User Requirements Stage: Use Case Diagram, Activity Diagrams, Deployment Diagram
    2. System Implementation Stage: Class Diagram, State Machine Diagram
    3. Development Stage: Use Case Diagram, Sequence Diagram, State Machine Diagram
    4. Validation and Testing Stage: State Transition Diagram
    5. Packaging and publishing Stage: Activity Diagram.

    Forum Discussion: Metamodel Design to support OOD of the IoT

    Object-Oriented Programming (OOP) is a foundational aspect of software development. Therefore, Fortino et al. (2015) introduces a new approach that aims to apply similar fundamentals to the Internet of Things (IoT). OOP initiatives require planning and design during early development stages. systematic software design strategies such as UML, helps provide a blueprint to support the development. Similarly, Metamodel is an approach that supports analysis and implementation techniques that aids with the development of Smart objects to the IoT (Fortino, et al., 2015). Metamodel approaches can strongly support and identify smart object functions in different abstraction levels. Early development phases must require efforts to understand main system design, relationships, and functional components. As such, metamodel approaches can provide abstract testing grounds for features in different phases of the system. Although system design planning approaches such as UML or Metamodel are supplemental factors of system development, it remains a challenge to include actual syntax within both approaches. That is because code components, functions, behaviours and algorithms are subject for constant change due to numerous factors during system implementation (Perez & Sellers 2008). A metamodel in concept provides a generalized approach of the system design. The inclusion of further implementation strategies can be time consuming and maybe inefficient.

    All references are recorded in unit 12.

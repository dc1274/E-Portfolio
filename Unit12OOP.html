Unit 11

Reflections & Evaluation (Assignment 1 System Implementation)

Following system design of the driverless car, is system implementation. My main goal was to be able to organize the code in several blocks which will aid in the production of a functional code. The first step was to build the base classes for every system component, which started with a base class for the main control unit that contained all the necessary features. The structure of the syntax was as follows:

Base Classes/ Abstract Classes: A blueprint for the child classes that will be created.

Main Classes: Classes to control the cars, record and update information. V2V Communisations, Traffic sign and obstacle inputs

permanent Objects: Deployed objects that contains the information which will be communicated through all car components/classes.

User Interface: Defined functions for user navigation through the control unit. (Driving the car, Inserting Objects, View Car information)

Several databases/structures were created to store the driver information, car types, object types and traffic sign types. Among one the important features were the car log which recorded all the driver’s actions. External libraries were added to the system for abstract class decorators and the exit functions to terminate the process upon the driver’s command.
